## Step 1: 内容提取 — 详细执行手册

**输入**：用户提供的3-5篇笔记（N篇）

**输出**：`./extracted_notes/note_N/`

---

### Step 1.1: 标题提取

**提取维度**：
- 结构类型（疑问型/陈述型/悬念型/清单型等）
- 字数统计
- 悬念手法（设问/反问/省略号/数字等）
- 核心要素（领域词/关键词/情绪词）

**输出格式**：
```markdown
## 标题分析

**结构类型**：{类型}
**字数**：{N}字
**悬念手法**：{手法描述}
**核心要素**：领域={}，关键词={}，情绪={}
```

---

### Step 1.2: 图片内容文案倒推（逐图）

**重要认知**：小红书笔记的图片结构是灵活的，不固定几张：
- 可能只有封面页
- 可能只有内容页（多页组合）
- 可能封面+内容页+尾页组合
- 具体几张，取决于内容稿子怎么安排

对笔记中的每张图片，先判断其用途（封面/内容页/尾页），再用7模块格式反推文案内容：

**7模块格式**：
| 模块 | 提取内容 |
|:---|:---|
| 1. 核心主题与用途 | 画什么、给谁看、用在哪 |
| 2. 风格与质感 | 基础风格+质感细节 |
| 3. 布局与构图 | 分栏/分区、视觉流 |
| 4. 配色方案 | 主色调、辅助色、强调色 |
| 5. 核心元素与细节 | 人物、物体、文字 |
| 6. 氛围与情绪 | 传递的感受 |
| 7. 技术与合规 | 分辨率要求、风险提示位置 |

**输出格式**：
```markdown
## 图片{N}（{用途：封面/内容页X/尾页}）

**图片用途判断**：{描述为什么判断为该用途}

**模块1-核心主题与用途**：{提取内容}
**模块2-风格与质感**：{提取内容}
**模块3-布局与构图**：{提取内容}
**模块4-配色方案**：{提取内容}
**模块5-核心元素与细节**：{提取内容}
**模块6-氛围与情绪**：{提取内容}
**模块7-技术与合规**：{提取内容}
```

---

### Step 1.3: 正文结构提取

**提取内容**：
- 内容段落分析（每段功能：引入/展开/总结）
- 固定引导识别：开头钩子、结尾CTA、风险提示位置与措辞
- 话术特征（人称/语气/句式）

**输出格式**：
```markdown
## 正文结构分析

**段落结构**：
| 段落 | 功能 | 字数 | 核心内容 |
|:---|:---|:---|:---|
| 第1段 | 引入 | N | {描述} |
| 第2段 | 展开 | N | {描述} |

**开头钩子**：{类型} - "{原文片段}"
**结尾CTA**：{类型} - "{原文片段}"
**风险提示**：{原文}

**话术特征**：
- 人称：{第一/第二/第三人称}
- 语气：{口语/专业/亲切/严肃}
- 句式特点：{短句为主/长句拆解/善用标点}
```

### Step 1.4: 绘画提示词倒推（逐图）

**重要原则**：每个模块必须提取**具体描述**，不能用抽象占位符。禁止使用"相关图标"、"等元素"、"等物体"等模糊表达。

对每张图片反推完整AI绘图提示词（7模块整合适式）：

**必须逐项具体描述，禁止模糊**：
- **人物**：数量、外貌特征（戴什么帽子/穿什么衣服/什么发型）、动作、表情
- **物体**：具体物品名称（不是"图标"，是"电池图标"、"金色金币"）、数量、位置
- **文字**：标题具体内容、气泡内具体文字、标签文字

**示例（正确示范）**：
```
【主题与用途】AI算力与储能的投资科普，用于小红书，受众为普通投资者
【风格与质感】卡通手绘风，暖米色底色，线条柔和，人物Q版活泼
【布局与构图】左右分栏，左侧为淘金痛点场景，右侧为"卖铲子"解决方案
【配色方案】主色调暖米色（#F5F5DC），辅助色浅棕（#D2B48C），强调色明黄（#FFD700）
【核心元素与细节】
- 人物：左侧3位戴安全帽的淘金矿工，手持淘金盘，表情兴奋；右侧1位手持写有"电力"字样铲子的男性
- 物体：AI火箭、金矿图标、储能电池、轮船（出海）、增长折线图、金币
- 文字：标题"投资，有时候要找到那个'卖铲子'的人"；气泡"大模型淘金！"、"AI应用热！"
【氛围与情绪】轻松科普，传递理性投资逻辑
【技术与合规】分辨率1080x1440（3:4竖版）；底部标注："市场有风险，投资需谨慎"
```

### ⚠️ 画图提示词固定部分（所有图片必须包含）

**以下部分是固定的，所有图片提示词都必须包含，不是提取的**：

1. **比例/分辨率**：固定为3:4竖版，分辨率固定为1080x1440
2. **风险提示**（仅内容页/总结页需要，封面可选）：固定格式："市场有风险，投资需谨慎 根据公开信息整理，不构成投资建议，图片含AI生成"，位置：图片底部
3. **禁止项**（固定不变）：禁止个股推荐、具体点位预测；禁止绝对化表述；禁止血腥/暴力/低俗内容

## Step 2: 分析归纳 — 详细执行手册

### Step 2.1: 写作风格 + 格式要求归纳

**来源**：所有笔记的文案结构分析

**提取内容**（必须完整，包含所有11个维度）：

1. **标题风格**（类型偏好/字数区间/悬念模式）
2. **正文结构**（段落数/每段功能/衔接方式）
3. **语体特征**（人称/语气/句式/标点习惯）
4. **固定结尾**（风险提示格式/署名格式/品牌口号）
5. **话题标签**（数量/类型/排列规则）
6. **段落强制开头语**：从样本中提取每个段落的固定开头语（如"走势结构上"、"鱼群方面"），**必须**标注哪些段落必须有固定开头语
7. **比喻体系**：样本中使用的特色比喻词库，**必须**标注哪些比喻是必须使用的
8. **禁止/允许清单**：从样本中提取的硬性边界
9. **图片使用偏好**：图片数量偏好/封面使用偏好/尾页使用偏好/内容页数量偏好/图片组合模式
10. **封面页结构**：标题形式/核心观点呈现方式/字数范围/是否包含副标题
11. **尾页结构**：内容组成/排列顺序/风险提示完整格式/署名格式/CTA类型

**品牌名、品牌口号、固定结尾模板**是本步骤的**最高优先级提取项**，必须从样本中精确提取。

**完整输出格式**：
```markdown
# 写作风格指南

## 品牌信息（最高优先级）
**品牌名**：{精确提取，如"年年有余"}
**品牌口号**：{原文}
**固定结尾模板**：{完整原文，包括所有标点和格式}

## 标题风格
**类型偏好**：{类型} | **字数区间**：{min}-{max}字 | **悬念模式**：{模式}

## 正文结构
**段落数**：{N}段 | **每段功能**：{描述} | **衔接方式**：{方式}

### 段落强制开头语
| 段落位置 | 必须使用的开头语 |
|:---|:---|
| 第1段 | {如无固定开头语则填"无"} |
| 第2段 | {如"近期事件层面"} |

## 比喻体系
**核心比喻**：
- **{比喻词}**：指代{含义}（必须使用）
- **{比喻词}**：指代{含义}（必须使用）

## 语体特征
**人称**：{人称} | **语气**：{语气} | **句式特点**：{特点}

## 固定结尾
**风险提示**：{原文模板}
**署名格式**：
```
作者：{姓名}
证券投顾执业资格编号：{编号}
单位：{营业部}
```

## 话题标签
**数量**：{N}个 | **排列规则**：{规则}

## 禁止清单
**严禁使用**：{词汇列表}

## 图片使用偏好
**图片数量偏好**：{固定N张/不固定/最少N张}
**图片组合模式**：{如"封面+内容+尾页"}

## 封面页结构
**是否存在封面页**：{是/否} | **标题形式**：{与正文标题相同/不同/简化版}

## 尾页结构
**是否存在尾页**：{是/否} | **内容组成**：{总结/风险提示/署名/CTA}
```

## Step 3: Skill组装 — 详细执行手册

---

### Step 3.1: 生成专属Skill主文件（SKILL.md）

**品牌名是最高优先级**：成品Skill的name、description、目录名、文件中所有出现"品牌名"的地方，**必须**使用Step 2.1精确提取的{品牌名}，不得自行创作或简化。

**必须包含的模块**：

**模块A - 热点检索**：
```markdown
## 热点检索模块
主动检索当日/本周财经热点：
- 财联社电报（P0）
- 东方财富快讯（P1）
- 同花顺热帖（P2）
```

**模块B - 合规限制**：
```markdown
## 合规红线
- 🚫 红线：个股推荐、具体点位、绝对化表述
- ✅ 必须项：风险提示 + 投顾署名

## 必须包含的固定结尾
{从固化模块复制：references/固化模块.md}
```

**模块C - 选题模块**（来自 `topic_analysis.md`）

**模块D - 写作模块**（来自 `style_guide.md`）
- 品牌名 + 品牌口号（最高优先级）
- **段落强制开头语**（每段必须使用的固定开头语）
- **比喻体系**（必须使用的比喻词）
- **禁止/允许清单**

**模块E - 绘画提示词模块**（来自 `image_style.md`）

**成品Skill主干流程**：
```
用户输入 → 选题判断 → 素材检索 → 标题生成 → 正文撰写 → 绘画提示词生成 → 合规校验
```

**【必须执行】从工厂固化资源池复制必要文件到成品Skill**：

- `references/固化模块.md` → 合规红线
- `references/creativity_rules.md` → 爆款规律库

### Step 3.5: 图片风格预设选择

**【仅当用户提供图片时执行；当用户无法提供图片时直接使用预设】**

#### 分支A：用户提供了图片
```
使用Step 1提取的图片风格 → 对比6种预设找最匹配的风格 →
以预设作为锚点优化反推结果 → 生成专属画图提示词
```

#### 分支B：用户无法提供图片
```
询问用户选择预设风格（6选1）：
1. professional（专业深蓝金）
2. fresh（清新粉蓝）
3. tech（科技紫霓虹）
4. warm（温暖橙黄）
5. minimal（简约黑白）
6. nature（自然绿棕）
```

### 4.2 格式评审

| 检查项 | 状态 |
|:---|:---:|
| frontmatter | ✅/❌ |
| 流程清晰 | ✅/❌ |
| 无占位符 | ✅/❌ |

---

### 4.3 内容测试

1. 从原始输入案例中随机选1篇
2. 倒推该案例的选题主题
3. 用该主题调用成品Skill生成
4. 对比输出与原始案例

| 维度 | 检查要点 | 阈值 |
|:---|:---|:---:|
| 品牌名一致性 | 成品输出中品牌名是否与提取一致 | 必须一致 |
| 完整性 | 选题概要/文稿/绘画提示词是否齐全 | 必须完整 |
| 标题风格 | 类型/长度/悬念手法一致性 | ≥80% |
| 正文结构 | 段落数/句式/语气一致性 | ≥80% |
| 段落开头语 | 固定开头语是否被使用 | ≥80% |
| 比喻体系 | 核心比喻是否被使用 | ≥80% |
| 画风一致性 | 配色/构图/元素一致性 | ≥80% |

---

### 4.4 修复循环

如Step 4.1、4.2或4.3未通过：
1. 定位问题来源文件
2. 修改对应内容
3. 重新执行审核

**最大循环次数**：3轮

---

### 4.5 输出确认

1. **确认输出目录**：询问用户成品Skill应输出到哪个目录（默认：`./output/{品牌名}-xhs/`）
2. **安装判断**：检查目标目录是否已存在同名Skill → 询问用户是否覆盖或重命名
3. **最终输出**：列出文件清单，确认安装状态
