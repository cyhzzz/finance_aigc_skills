# 小红书卡片规范

> 本文档是多平台改写流程 Step 6.2 的详细操作手册

---

## 小红书笔记输出要求

### 格式要求

| 要求 | 说明 |
|-----|------|
| 文件格式 | Markdown（.md） |
| 段落 | 短段落，每段 ≤ 3句 |
| emoji | 大量使用，每段至少1个 |
| 空行 | 段间空行分隔 |
| 标签 | 5-8个话题标签（#开头） |
| 配图 | 3:4 竖版，3-6张提示词 |

---

## 小红书笔记结构

```markdown
# {吸引眼球的标题}

{正文第1段：引入+emoji}
（空行）
{正文第2段：核心内容+emoji}
（空行）
{正文第3段：要点+emoji}
（空行）
{正文第4段：总结+emoji}
（空行）
{正文第5段：行动引导+emoji}

{标签1} {标签2} {标签3} {标签4} {标签5}
```

## 标题规范

### 标题类型

| 类型 | 示例 |
|-----|------|
| 疑问句型 | 还在看不懂大盘？这个方法3分钟学会 |
| 数字型 | 2024年股市最重要的3个信号 |
| 情绪型 | 终于有人说清楚了！ |
| 悬念型 | 这个板块可能要爆发了... |

### 标题规则

- 字数：15-25字
- 必须含情绪词或悬念元素
- 必须含关键词（板块/概念/工具）
- 避免标题党（过度承诺）

---

## 正文规范

### 段落规则

- 每段 ≤ 3句
- 段首可使用 emoji 引导
- 段末可使用 emoji 加强
- 重要信息用 `**加粗**` 标注

### emoji 使用指南

| 场景 | emoji |
|-----|------|
| 引入 | 🔍 💡 📊 🎯 |
| 强调 | ⚠️ ❗ 💰 🚀 |
| 情绪 | 😱 🤯 💪 🎉 |
| 收尾 | 👇 📝 🔥 |
| 数据 | 📈 📉 📊 |

### 常用句式

```
开头：
- 💡 什么是XXX？3分钟给你讲清楚
- 📊 今天聊聊XXX，这可能是今年最重要的...
- 🎯 XXX到底怎么用？看这一篇就够了

结尾：
- 👇 有问题评论区见
- 📝 觉得有用记得收藏
- 🔥 关注我，持续更新...
```

## 标签规范

### 标签规则

- 数量：5-8个
- 首位标签：# + 核心关键词
- 末位标签：#投顾建议 / #投资有风险
- 中间标签：热点词 + 工具词 + 场景词

### 标签示例

```
#A股 #市场分析 #基金定投 #理财入门 #投资有风险 #投顾建议
```

## 配图提示词规范

### 图片数量

- 封面图：1张
- 内容图：2-5张
- 总计：3-6张

### 3:4竖版提示词结构

```markdown
### 图1（封面图）
[3:4竖版，画面主体为{描述}，风格{风格}，配色{配色}，含标题文字"{标题}"于画面{位置}]

### 图2（内容图）
[3:4竖版，画面主体为{描述}，风格{风格}，配色{配色}]

### 图3（内容图）
[3:4竖版，画面主体为{描述}，风格{风格}，配色{配色}]
```

### 提示词要求

每个提示词必须包含：
1. **画面主体**：具体描述（人物/物体/场景）
2. **风格**：简约/可爱/商务/科技感
3. **配色**：主色调
4. **构图**：左右分栏/上下布局/居中
5. **文字叠加**（仅封面）：标题文字位置

### 风险提示（内容图/总结图）

图片底部固定添加：
```
底部文字：市场有风险，投资需谨慎
```

## 小红书卡片 HTML

### 卡片 HTML 结构

```html
<div style="width: 320px; background: #fff; border-radius: 8px; overflow: hidden; box-shadow: 0 2px 8px rgba(0,0,0,0.1);">
  <!-- 图片区域 -->
  <div style="width: 100%; aspect-ratio: 3/4; background: #f5f5f5; display: flex; align-items: center; justify-content: center;">
    <p style="color: #999; font-size: 12px; text-align: center; padding: 20px;">[图片：{描述}]</p>
  </div>
  <!-- 文字区域 -->
  <div style="padding: 12px;">
    <h3 style="font-size: 14px; font-weight: bold; margin: 0 0 8px; line-height: 1.4;">{标题}</h3>
    <p style="font-size: 12px; color: #666; margin: 0; line-height: 1.5;">{核心金句}</p>
    <p style="font-size: 10px; color: #999; margin: 8px 0 0;">{来源}</p>
  </div>
</div>
```

### 卡片 HTML 用途

- 用于生成可复制的小红书风格卡片图片
- 可在生成后让用户自行截图使用
- 不依赖微信上传API

## 小红书合规要求

### 必须包含

- 风险提示（底部或文末）
- 数据来源标注
- 不承诺收益

### 禁止表达

- ❌ "一定涨"、"保证赚钱"
- ❌ "跟着买就能赚钱"
- ❌ 具体个股推荐
- ❌ "稳赚不赔"

## 渲染选项

生成小红书配图时，用户可选择两种模式：

### 模式 A：AI 绘图提示词（默认）

生成 AI 绘图提示词，调用图像生成 API（需配置 image api_key）。

适合：有 AI 绘图账号、追求风格多样化的团队。

```
用户：「发一篇关于 AI Agent 的小红书」
→ 3-6 张 AI 绘图提示词（3:4 竖版，风格/配色/构图指定）
→ 用户调用生图 → 手动拼接
```

### 模式 B：HTML → PNG 渲染（新增）

使用 ljg-card `-m` 模式将内容直接渲染为 PNG 小绿书卡片。

适合：无 AI 绘图账号、内容质量优先、追求真实感的个人博主。

```
用户：「发一篇关于 AI Agent 的小红书」
→ 内容文本 → 自动切分卡片（1080×1440）
→ 色调感知匹配气质（思辨蓝/技术灰/文学暖/科学绿）
→ 贪心切分保障阅读节奏
→ 渲染 HTML → Playwright 截图 → 3-6 张 PNG
```

**技术说明**：
- 依赖：`node` + `playwright`（`npx playwright install chromium`）
- 模板：`toolkit/assets/poster_template.html`
- 截图：`toolkit/assets/capture.js`
- 输出：`output/xhs/{name}_1.png`, `..._2.png` ...
- 字体：优先 KingHwa_OldSong，备选 Noto Serif SC
- 触发：平台风格手册中选择「XHS 卡片渲染模式 B」

**设计品味**：遵循 `references/taste.md` 准则——禁止 Inter 字体、禁止纯黑、禁止三等分卡片。
