# 内容创作框架 - 使用指南

## 概览

本框架提供通用内容创作的完整方法论，适用于各类内容创作场景。

---

## 快速开始

### 1. 理解核心理念

本框架的四大核心理念：

1. **认知重构**：挖掘需求背后的本质问题
2. **结构化思考**：用分析框架组织逻辑
3. **带立场**：有明确的观点和判断
4. **真实细节**：用具体性增强可信度

---

### 2. 掌握创作流程

六步创作流程：

```
Step 1: 需求理解与重构（⭐必须优先执行）
Step 2: 选题讨论（必做）
Step 3: 结构化设计
Step 4: 信息搜集与验证（⭐必须执行）
Step 5: 内容创作
Step 6: 质量检查（三遍审校）
```

---

### 3. 学习分析框架

五种常用框架：

| 框架 | 适用场景 | 核心要素 |
|------|---------|---------|
| **问题-原因-方案-升华** | 教程类 | 问题描述、原因分析、解决方案、价值升华 |
| **故事-冲突-转折-洞察** | 经验类 | 个人故事、冲突事件、转折点、核心洞察 |
| **颠覆认知-新视角-重建** | 观点类 | 挑战旧认知、提供新视角、重建认知 |
| **对比-反差-原因-启示** | 对比类 | 多个选项、反差对比、原因分析、启示 |
| **数据冲击-深挖-案例-升华** | 分析类 | 数据呈现、深挖分析、案例支撑、总结升华 |

## 使用场景

### 场景1：创作一篇教程类文章

**适用框架**：问题-原因-方案-升华

**执行流程**：

```
【Step 1: 需求理解】
用户："写一篇关于'如何提高工作效率'的教程"

智能体重构：
问题本质：用户感觉效率低，不知道如何提升
受众：知识工作者
核心信息：1. 砍掉无效工作 2. 优化流程 3. 接受不完美

【Step 2: 选题讨论】
提供3-4个选题方案，选择"工作效率的三个陷阱"

【Step 3: 结构化设计】
使用"问题-原因-方案-升华"框架

【Step 4: 信息搜集】
搜集效率提升的方法和案例

【Step 5: 内容创作】
按照结构创作内容

【Step 6: 质量检查】
三遍审校，检查质量标准
```

---

**适用框架**：颠覆认知-新视角-重建

**执行流程**：

```
【Step 1: 需求理解】

【Step 2: 选题讨论】
提供3-4个选题方案，选择"AI可以替我写，但不能替我想"

【Step 3: 结构化设计】
使用"颠覆认知-新视角-重建"框架

【Step 4: 信息搜集】
搜集相关研究和个人经历

【Step 5: 内容创作】
按照结构创作内容，加入个人经历和观点

【Step 6: 质量检查】
三遍审校，检查观点鲜明度和价值启发度
```

**适用框架**：对比-反差-原因-启示

```

问题本质：两个工具各有特点，用户不知道如何选择
受众：笔记工具用户
核心信息：Notion适合协作，Obsidian适合个人

【Step 2: 选题讨论】
提供3-4个选题方案，选择"笔记工具选择：Notion vs Obsidian"

【Step 3: 结构化设计】
使用"对比-反差-原因-启示"框架

【Step 4: 信息搜集】
搜集两个工具的详细信息和用户评价

【Step 5: 内容创作】
按照结构创作内容，给出明确建议

【Step 6: 质量检查】
三遍审校，检查建议的明确性
```

## 质量检查清单

### 内容质量检查

```
□ 结论是否先行？
□ 观点是否鲜明（有明确立场）？
□ 逻辑是否严密（无跳跃/矛盾）？
□ 细节是否具体（避免泛化表达）？
□ 是否有数据/案例支撑？
```

---

### 结构质量检查

```
□ 是否使用了分析框架？
□ 结构是否清晰（层次分明）？
□ 开头是否有价值承诺/悬念？
□ 结尾是否有延伸反问？
```

---

### 格式规范检查

```
□ 标题层级清晰
□ 列表使用正确（有序/无序）
□ 表格使用恰当（多维度对比）
□ 引用突出重点
□ 代码块指定语言
```

## 常见问题

### Q: 如何选择合适的分析框架？

**A: 根据创作类型选择**：

- 教程类 → 问题-原因-方案-升华
- 经验类 → 故事-冲突-转折-洞察
- 观点类 → 颠覆认知-新视角-重建
- 对比类 → 对比-反差-原因-启示
- 分析类 → 数据冲击-深挖-案例-升华

---

### Q: 如何让观点更鲜明？

**A: 遵循"结论先行"原则**：

- 开头第一段就给出核心观点
- 避免"各有优势"等中庸表达
- 用数据和案例支撑观点
- 说明观点适用的边界条件

---

### Q: 如何让内容更真实具体？

**A: 遵循"具体性原则"**：

- 时间：具体到"上周三"而非"最近"
- 地点：具体到"在星巴克"而非"在某地"
- 人物：具体到"和3个朋友"而非"一些人"
- 数据：精确数字而非"很多"

---

### Q: 如何进行三遍审校？

**A: 按照以下流程**：

**第一遍：基本错误检查**
- 错别字、标点、格式

**第二遍：表达和逻辑优化**
- 句式流畅、逻辑严密、过渡自然

**第三遍：风格和价值提升**
- 结论先行、结构清晰、观点鲜明、细节具体、有延伸

## 进阶技巧

### 技巧1：多轮迭代

创作不是一次性完成的，可以多轮迭代：

1. 第一轮：基于结构和框架快速成稿
2. 第二轮：补充细节和案例
3. 第三轮：打磨表达和节奏

---

### 技巧2：个性化适配

根据受众不同，调整创作风格：

- **专业受众**：更多技术细节，更简洁
- **新手受众**：更多解释和比喻，更详细
- **决策者**：更多数据和趋势，更少技术细节

---

### 技巧3：多格式输出

同一内容可以适配不同格式：

- 长文（公众号、博客）
- 短文（社交媒体）
- 演讲稿（线下分享）
- 视频脚本（短视频/长视频）

## 总结

高质量内容创作的核心是"思考"，而非"写作"。

本框架提供的不是模板，而是**方法论**：

1. **认知重构**：挖掘问题本质
2. **结构化思考**：用框架组织逻辑
3. **带立场**：有明确观点和建议
4. **真实细节**：用具体性增强可信度

最终目标：创作有价值、有深度、有启发的内容，让读者不仅得到信息，更获得思考。
