# Phase详细执行指南

本文档提供每个Phase的详细执行步骤、输出格式和注意事项。

---

## Phase 0: 模式判断 - 详细执行

### 任务目标
根据用户输入的内容长度和特征，自动判断执行创作模式还是改写模式。

### 判断逻辑

```markdown
## 模式判断

**输入内容长度**：<统计字数>

**判断结果**：
- 字数 > 200字 → 【改写模式】
- 字数 ≤ 200字 → 【创作模式】
```

### 改写模式特点

**输入**：用户提供已有文章/草稿/笔记（大段内容）
**任务**：
- 分析原文核心问题
- 提炼核心观点（1-3个）
- 优化逻辑结构
- 寻找投资转向机会
- 改写优化内容

**后续流程**：Phase 1 → Phase 1.5 → Phase 2

### 创作模式特点

**输入**：用户提供简短需求/话题/关键词
**任务**：
- 需求理解与重构
- 选题讨论（3-4个选题方案）
- 投资转向分析
- 结构化设计
- 从零开始创作

**后续流程**：Phase 1 → Phase 1.5 → Phase 2

### 输出格式

```markdown
## 模式判断结果

**输入内容长度**：<统计字数>

**判断结果**：【改写模式】/【创作模式】

**下一步**：进入 Phase 1 初稿创作
```

## Phase 1: 初稿创作 - 详细执行 ⭐ 第一阶段

### 任务目标
**纯粹的内容创作阶段**，专注于选题探讨、优质初稿创作，**不考虑IP风格和平台格式**。

### 核心原则
- ✅ 专注于内容质量和价值
- ✅ 深度探讨选题角度
- ✅ 创作优质内容初稿
- ❌ 不考虑账号风格
- ❌ 不考虑平台格式
- ❌ 不考虑具体表达方式

---

### 创作模式详细流程

#### Step 1.1: 需求理解与重构

**任务**：深度分析用户需求本质

**执行步骤**：
1. 分析用户输入的话题/关键词
2. 挖掘需求背后的本质问题
3. 识别目标受众和场景
4. 明确核心信息点（1-3个）

**输出格式**：
```markdown
## 需求分析

**用户输入**：<原始输入>

**本质问题**：<提炼的本质问题>

**目标受众**：<受众特征>

**核心信息**：
1. <核心点1>
2. <核心点2>
3. <核心点3>
```

#### Step 1.2: 投资引导分析 ✋ 必须确认

**任务**：分析选题与投资的关联，设计引导来申万宏源投资的切入点

**参考**：`references/investment-guidance-guide.md`

---

**核心原则**：
1. **自然过渡**：避免生硬转折，找到话题与投资的内在逻辑联系
2. **价值优先**：先提供价值，再进行引导
3. **软硬结合**：软性引导为主，硬性引导为辅

---

**分析内容**：

##### 1. 选题与投资的关联

分析这个话题如何自然转向投资理财，找到内在逻辑联系。

**常见关联类型**：

| 话题类型 | 投资关联 | 过渡逻辑 |
|---------|---------|---------|
| 美食/购物 | 消费升级、理财规划 | 省钱 → 理财 → 投资 |
| 旅游/出行 | 文旅板块、财务自由 | 旅行成本 → 提前规划 → 投资 |
| 职场/成长 | 财务规划、定投策略 | 收入增长 → 让钱增值 → 投资 |
| 科技热点 | 科技板块、行业ETF | 科技改变生活 → 科技投资机会 |
| 政策热点 | 政策受益板块 | 政策解读 → 普通人参与（投资工具） |

##### 2. 软性切入点设计

**知识科普型**：建立专业形象
```
很多人不知道，ETF其实是一种非常适合新手的投资工具...
作为老牌券商，申万宏源有30年的投资研究积累...
```

**价值渗透型**：让用户意识到投资价值
```
假设你现在22岁，每个月定投1000块，按年化8%计算...
这就是时间和复利的力量。
```

**案例启发型**：通过成功案例激发向往
```
我有个朋友，从3年前开始每月定投指数基金...
3年下来，账户已经有12万了。
```

**避坑指南型**：通过揭示风险建立信任
```
新手最容易犯的3个错误...
后来在申万宏源投顾的帮助下才建立了正确的投资体系。
```

##### 3. 硬性切入点设计

**新客权益型**：
```
🎁 8.18%新客专属理财（5万元起投，30天）
📊 A股Level-2行情免费体验3个月
📈 3000元投顾服务代金券
```

**开户引导型**：
```
为什么选择申万宏源？
✅ 成立30年的老牌券商
✅ 全业务牌照，一站式服务
✅ 全国200+营业部
```

**资料领取型**：
```
我们整理了3份资料：
📄 ETF新手入门大全
📊 定投实操指南
💼 适合自己的投资方法
```

##### 4. 过渡逻辑设计

**递进关系**：问题 → 后果 → 解决方案（投资）
```
职场新人收入不高（问题）
→ 想提高收入但没方向（后果）
→ 投资理财是让钱增值的好方法（解决方案）
```

**对比关系**：常规做法 → 局限性 → 更好的方法
```
只顾工作赚钱 → 钱放银行贬值
→ 让钱通过投资增值（更好的方法）
```

**时间维度**：现在 → 未来影响 → 优化未来
```
现在省下的钱 → 10年后可能贬值
→ 通过投资让财富增值
```

**类比关系**：熟悉场景 → 投资场景 → 投资方法
```
就像种树需要选种子、耐心等待
投资也需要选好标的、长期持有
```

##### 5. 引导策略选择

**纯软性策略**（知识科普类、建立信任）：
- 90% 知识价值
- 5% 品牌露出
- 5% 资料领取

**软硬结合策略**（大部分笔记，平衡内容和推广）：
- 70% 知识价值
- 20% 软性引导
- 10% 硬性推广

**硬性策略**（广告笔记、权益推广）：
- 40% 知识价值
- 60% 硬性推广

#### Step 1.4: 内容逻辑结构设计 ✋ 必须确认

**任务**：使用分析框架组织内容的**逻辑结构**

---

#### ⚠️ 重要区分：逻辑结构 vs 小红书格式

**Step 1.4（本步骤）**：内容的**逻辑结构**
- 这是文章本身的**思维逻辑**
- 例如：问题 → 原因 → 方案 → 升华
- 目的：让内容有逻辑、有深度

**Phase 2（后续步骤）**：小红书的**展示格式**
- 将内容拆分为多页笔记
- 例如：封面页 → 内容页1 → 内容页2 → ... → 尾页
- 目的：适配小红书平台，方便用户阅读

**区别示例**：

| 维度 | Step 1.4 逻辑结构 | Phase 2 小红书格式 |
|------|-------------------|------------------|
| **性质** | 内容的思维逻辑 | 内容的展示形式 |
| **关注点** | 逻辑严密、论证充分 | 视觉友好、易于阅读 |
| **示例** | 问题-原因-方案-升华 | 封面+5页内容+尾页 |
| **受众** | 内容创作者 | 小红书用户 |
| **执行时机** | Phase 1 - 初稿创作 | Phase 2 - 格式适配 |

**类比理解**：
- 逻辑结构 = 电影的剧本结构（开端、发展、高潮、结局）
- 小红书格式 = 电影的分镜头呈现（多个场景、特写、配乐）

#### 常用分析框架

**问题-原因-方案-升华**（教程类）
```
问题：职场新人收入不高
原因：单一收入来源，抗风险能力弱
方案：投资理财，让钱增值
升华：实现财务自由，生活更有选择
```

**故事-冲突-转折-洞察**（经验类）
```
故事：我的投资经历
冲突：亏损10万的教训
转折：改变投资策略
洞察：投资本质是认知变现
```

**颠覆认知-新视角-重建**（观点类）
```
颠覆：打工能致富？错
新视角：让钱为你工作才是真相
重建：建立正确的投资观念
```

**对比-反差-原因-启示**（对比类）
```
对比：存银行 vs 投资基金
反差：收益差距巨大
原因：通货膨胀吞噬存款
启示：必须学会投资理财
```

**数据冲击-深挖-案例-升华**（分析类）
```
数据：90%的人投资亏损
深挖：原因是什么？
案例：成功的投资者怎么做
升华：投资是一门可以学习的技能
```

## Phase 1.5: 审校优化 - 详细执行 ⭐ 关键步骤

### 任务目标
对初稿进行正式的审校和优化，确保内容质量达标。

### 审校维度

#### 1. 内容准确性
- 事实是否准确
- 数据是否正确
- 逻辑是否严密
- 有无错误信息

#### 2. 表达清晰度
- 语言是否清晰易懂
- 句子是否通顺
- 表达是否准确
- 有无歧义表达

#### 3. 结构合理性
- 逻辑是否清晰
- 结构是否完整
- 过渡是否自然
- 层次是否分明

#### 4. 价值密度
- 信息是否充分
- 观点是否有价值
- 细节是否具体
- 是否有启发性

### 优化内容

根据审校结果进行优化：

1. **润色文字表达**
   - 优化句子结构
   - 提升表达精准度
   - 增强可读性

2. **优化逻辑结构**
   - 调整段落顺序
   - 优化过渡句
   - 强化逻辑关联

3. **补充必要细节**
   - 添加具体案例
   - 补充数据支撑
   - 增加细节说明

4. **提升可读性**
   - 简化复杂表达
   - 增加解释说明
   - 优化段落长度

### 输出格式

```markdown
## 审校优化报告

### 审校结果

**内容准确性**：✅/⚠️
**表达清晰度**：✅/⚠️
**结构合理性**：✅/⚠️
**价值密度**：✅/⚠️

### 优化说明

<具体进行了哪些优化>

### 优化后内容

<审校优化后的完整内容>

**核心观点**：<1-2句话总结>

**字数**：<统计字数>

**下一步**：进入 Phase 2 IP风格迁移 + 平台格式适配
```

### 确认要求
- ✅ 展示审校后内容
- ✅ 说明优化要点
- ✅ 等待用户确认

## Phase 2: IP风格迁移 + 平台格式适配 - 详细执行 ⭐ 第二阶段

### 任务目标

### 核心原则

- ✅ 转换为选定账号风格
- ✅ 适配小红书平台格式
- ✅ 添加风险提示和推广

### Step 2.1: 账号风格选择

#### 任务目标
明确是为哪个小红书账号撰写笔记，确保内容符合账号定位。

#### 6个账号风格对比表

| 账号名称 | 类型 | 风格特征 | 适用场景 | 语言风格 |
|---------|------|---------|---------|---------|
| **小红书主号-投教笔记** | 官方账号 | 理财教育导向，亲切口语化 | 知识科普、理财入门 | "攻略""秘籍""宝子们" |
| **小红书主号-广告笔记** | 官方账号 | 开户推广导向，利益吸引 | 新客权益、开户引导 | "薅羊毛""手慢无""限时" |

### 改写模式执行流程

#### Step 1: 需求理解与重构

**分析用户输入**：
- 识别原文的核心问题
- 明确目标受众是谁
- 提炼核心信息点（1-3个关键点）

**重构需求**：
- 将用户输入转化为明确的工作任务
- 识别可以融入的投资转向要素

#### Step 2: 内容分析

**提取核心观点**：
- 找出原文最重要的1-3个观点
- 理清观点之间的逻辑关系

**分析逻辑结构**：
- 识别原文的结构框架
- 找出逻辑跳跃或不连贯的地方

**识别关键素材**：
- 标记重要数据
- 找出有价值案例
- 识别可引用的素材

#### Step 3: ⭐ 投资转向机会分析（核心步骤）

**读取指南**：`references/investment-pivot-guide.md`

**转向策略选择**：

1. **理财观念转向**：从消费观念转向理财观念
   - 适用：购物、消费、生活方式类话题
   - 示例：美食话题 → "会吃也要会理财"

2. **财务安全转向**：从风险管理角度切入
   - 适用：职场、收入、生活规划类话题
   - 示例：职场话题 → "收入上涨，财务安全跟上"

3. **机会成本转向**：从成本收益角度分析
   - 适用：教育、投资、决策类话题
   - 示例：教育话题 → "教育的投资回报率"

4. **资产配置转向**：从资源配置角度展开
   - 适用：宏观经济、市场趋势类话题
   - 示例：旅游话题 → "旅游也是一种资产配置"

5. **财富增长转向**：从财富积累角度引导
   - 适用：创业、副业、收入增长类话题
   - 示例：副业话题 → "多渠道收入，财富加速"

**设计软性引导路径**：
1. 从原文话题自然切入
2. 找到与投资理财的连接点
3. 用类比或案例过渡
4. 引入投资理财知识
5. 软性引导开户/咨询

**检查清单**：
- [ ] 转向是否自然？
- [ ] 是否有知识价值？
- [ ] 是否不是硬广？
- [ ] 引导是否软性？

#### Step 4: 改写优化

**优化逻辑结构**：
- 调整段落顺序
- 添加过渡句
- 强化逻辑连接

**提升表达清晰度**：
- 简化复杂句式
- 消除歧义表达
- 统一术语使用

**融入投资转向要素**：
- 在合适位置插入投资转向段落
- 确保转向与原文内容融合
- 添加数据或案例支撑

**保持核心观点不变**：
- 核心观点必须保留
- 可以优化表达方式
- 可以补充论据

#### Step 5: 质量检查（三遍审校）

**第一遍：基本错误检查**
- 错别字
- 标点符号
- 格式规范

**第二遍：表达和逻辑优化**
- 句式是否流畅
- 逻辑是否连贯
- 表达是否准确

**第三遍：风格和价值提升**
- 投资转向是否自然
- 知识价值是否足够
- 是否符合目标受众需求

## Phase 2.5: 爆款潜力评分与优化建议 - 详细执行

### 任务目标
使用 `xiaohongshu-viral-analyzer` skill对Phase 2输出的小红书笔记进行爆款潜力评估。

### 读取skill

**Skill名称**：`xiaohongshu-viral-analyzer`

**集成位置**：`skills/xiaohongshu-viral-analyzer/SKILL.md`

### 评估维度：高密四维结构

#### 1. 极致对标（25分）

**评估要点**：
- 是否重组了经过市场验证的成功要素
- 是否有清晰的赛道定位
- 是否有跨界降维的创新

**评分标准**：
- 20-25分：精准对标爆款要素，有创新
- 15-19分：有对标但创新不足
- 0-14分：缺乏对标，原创焦虑

#### 2. 情绪放大（25分）

**评估要点**：
- 是否直击人性（恐惧/贪婪/虚荣/色欲）
- 是否制造反差、阶层焦虑或爽感
- 是否每300字出可截图金句

**评分标准**：
- 20-25分：强情绪刺激，金句频出
- 15-19分：有情绪但不够强烈
- 0-14分：平铺直叙，缺乏情绪

#### 3. 算法迎合（25分）

**评估要点**：
- 标题是否有视觉锤，0.5秒内拦截手指
- 是否预埋槽点骗取互动和推荐
- 是否SEO友好，关键词布局合理

**评分标准**：
- 20-25分：强算法迎合，互动陷阱明显
- 15-19分：有一定算法考虑
- 0-14分：忽视算法，纯内容导向

#### 4. 工业结构（25分）

**评估要点**：
- 是否符合标准模组（钩子→痛点→方案→召唤）
- 黄金三秒：首句是否定生死
- 正文是否短句分割，视觉降噪

**评分标准**：
- 20-25分：完美SOP结构，节奏紧凑
- 15-19分：结构基本完整
- 0-14分：结构混乱，缺乏逻辑

### 输出要求

- **综合评分**：0-100分
- **四维得分**：每项0-25分
- **优化建议**：≤100字，具体可操作，使用互联网黑话降维指导

### 输出格式

```markdown
## Phase 2.5: 爆款潜力分析报告

### 综合评分
<总分>/100分

### 四维得分
- 极致对标：<得分>/25分
- 情绪放大：<得分>/25分
- 算法迎合：<得分>/25分
- 工业结构：<得分>/25分

### 优化建议（≤100字）
<简洁的改进建议，指出1-2个关键提升点>
```

### 优化建议参考模板

**低分场景（<60分）**：
```
颗粒度太散，缺乏情绪钩子。建议：
1) 标题加入数字/恐吓型强钩子
2) 每300字植入争议观点或金句
3) 预埋槽点（如"你们觉得对吗？"）骗评论
```

**中分场景（60-79分）**：
```

1) 制造反差或焦虑，放大痛点
2) 强化首句钩子，控制跳出率
3) 增加"90%的人不知道"类社会证明
```

**高分场景（80+分）**：
```

2) 优化封面图视觉冲击力
```

## Phase 3: 多页信息图生成 - 详细执行

### 任务目标
使用 `baoyu-xhs-images` skill为**每一页笔记**生成对应的小红书风格图文系列。

**核心特点**：
- **不是生成单张总览图**
- **为每页笔记生成对应图片**
- 封面页1张 + 内容页N张 + 尾页1张
- 所有图片保持风格统一

### 读取指南

**文件位置**：`references/phase3-multi-image-guide.md`

**读取该指南的**：
- 封面页设计规范
- 内容页设计规范
- 尾页设计规范
- 风格统一性要求

### 读取子skill配置

**Skill名称**：`baoyu-xhs-images`

**集成位置**：`skills/baoyu-xhs-images/SKILL.md`

**备用位置**：`~/.claude/skills/baoyu-xhs-images/SKILL.md`

**读取该skill的**：
- 10种视觉风格配置
- 8种布局方式
- 3种内容策略
- 首次设置偏好（EXTEND.md）

### 执行流程

`baoyu-xhs-images` skill 有自己的完整工作流程（Step 0-6）。本 Phase 将集成该 skill 的流程。

#### Step 0: 加载偏好设置（EXTEND.md）

skill 会自动检查用户偏好配置：
- 如未配置，引导用户完成首次设置
- 如已配置，加载偏好（风格、布局、水印等）

#### Step 1: 分析笔记图内容

从 Phase 2 输出中提取笔记图部分进行深度分析：
- 内容类型分类（干货/测评/教程/避坑等）
- 目标受众识别
- 视觉机会映射
- 推荐图片数量（2-10张）

#### Step 2: 确认 1 - 内容理解

用户确认核心卖点、目标受众、风格偏好。

#### Step 3: 生成 3 种策略方案

**Strategy A - 故事驱动型**（情感共鸣）
- 适合真实分享、个人体验
- 推荐风格：warm / cute / fresh

**Strategy B - 信息密集型**（价值优先）
- 适合教程、对比、测评
- 推荐风格：notion / minimal / chalkboard

**Strategy C - 视觉优先型**（氛围感）
- 适合高审美产品、生活方式
- 推荐风格：bold / pop / retro

每个策略包含：页数 + 大纲 + 推荐风格

#### Step 4: 确认 2 - 方案、风格、元素选择

用户选择：
1. 大纲策略（A/B/C 或组合）
2. 视觉风格（10种可选）
3. 视觉元素（背景、装饰、强调）

#### Step 5: 生成图像

为每页笔记生成对应的小红书风格图文：
- 封面页（吸睛标题 + 核心观点）
- 内容页（N张，充分展开）
- 尾页（引导开户 + CTA）

#### Step 6: 完成报告

输出生成结果和文件清单。

### 输出格式

`baoyu-xhs-images` skill 会生成完整的报告：

```markdown

Topic: [话题]
Strategy: [A/B/C/Combined]

Location: [目录路径]
Images: N total

Files:
- 01-cover-[slug].png ✓ Cover (sparse)
- 02-content-[slug].png ✓ Content (balanced)
- 03-content-[slug].png ✓ Content (dense)
- ...
- NN-ending-[slug].png ✓ Ending (sparse)
```

**session 目录**：`xhs-images/{topic-slug}/`

**包含文件**：
- analysis.md（内容分析）
- outline-strategy-a/b/c.md（三种策略）
- outline.md（最终选定的方案）
- prompts/*.md（每页的生成提示词）
- 01-cover.png, 02-content.png, ..., NN-ending.png（生成的图片）

## 附录：常见Phase相关问题

### Q: Phase 1文本量不够800字怎么办？

**A**:
- 检查是否充分展开了每个要点
- 补充更多案例和数据
- 增加投资转向部分的深度
- 扩展实用建议部分

### Q: Phase 2改写后核心观点变了怎么办？

**A**:
- 重新执行Phase 2
- 强调"保持核心观点不变"的要求
- 对比原文和改写稿，找出变化点
- 修正改写稿，确保核心观点一致

### Q: Phase 2.5评分低于60分怎么办？

**A**:
- 仔细阅读优化建议
- 针对短板维度进行优化
  - 极致对标低：增加爆款要素
  - 情绪放大低：增加情绪刺激
  - 算法迎合低：优化标题和槽点
  - 工业结构低：调整结构和节奏
- 优化后重新评分

### Q: Phase 3生成的图片风格不统一？

**A**:
- 检查是否严格遵守了固定参数
- 重新生成时强调风格一致性
- 使用相同的风格参数生成所有图片
- 确保所有提示词包含相同的风格描述

### Q: 用户在确认点要求大幅修改怎么办？

**A**:
- 重新执行当前Phase
- 融入用户的修改意见
- 再次确认后再继续
- 必要时回溯到之前的Phase
