## 1. content-creation-framework（内容创作框架）

### Skill名称
`content-creation-framework`

### 路径配置
见 [sub-skills-config.md](sub-skills-config.md)

### 功能
提供从需求理解、选题讨论、结构化设计到质量检查的完整内容创作方法论。

### 核心流程

#### 改写模式下使用
- **Step 1: 需求理解与重构**
  - 分析用户输入的大段内容
  - 提取核心问题、受众、关键信息
- **Step 2: 内容分析**
  - 提取核心观点
  - 理清逻辑结构
  - 识别关键数据和案例
- **Step 3: 改写优化**
  - 优化逻辑结构
  - 提升表达清晰度
  - **保持核心观点不变**
- **Step 6: 质量检查**
  - 三遍审校确保质量

#### 创作模式下使用
- **Step 1: 需求理解与重构**（必须优先）
- **Step 2: 选题讨论**（提供3-4个选题）
- **Step 3: 结构化设计**（选择分析框架）
- **Step 4: 信息搜集与验证**（如需要）
- **Step 5: 内容创作**（按结构创作）
- **Step 6: 质量检查**（三遍审校）

### 输出格式
改写模式：核心观点 + 优化后内容 + 改动说明
创作模式：需求理解 + 选定选题 + 完整初稿

### 何时读取
Phase 1开始前读取，了解完整的创作方法论

## 2. xiaohongshu-finance-writer（小红书财经笔记）

### Skill名称
`xiaohongshu-finance-writer`

### 路径配置
见 [sub-skills-config.md](sub-skills-config.md)

### 功能
通用小红书财经笔记改写工具，支持多账号风格动态切换。根据Phase 0.5中确认的账号风格，将内容改写为符合该账号定位的小红书笔记稿。

### 核心特征

#### 支持的账号风格
1. **小红书主号-投教笔记**（申万宏源证券财富管理）- 官方账号，理财教育导向
2. **小红书主号-广告笔记**（申万宏源证券财富管理）- 官方账号，开户推广导向
3. **叮叮猫的财经笔记** - KOS账号，金融知识科普，小白友好
4. **证券人排雷哥笔记** - KOS账号，系统化知识科普
5. **申万宏源乐乐笔记** - KOS账号，热点驱动型科普
6. **仿中金公众号文风** - 公众号风格，理性分析

#### 动态配置机制
- **读取配置**：根据Phase 0.5确认的账号风格，动态读取对应的配置文件
- **应用风格**：自动应用该账号的语言风格、内容结构、推广模板
- **风格一致性**：确保输出与选定账号风格保持完全一致

#### 篇幅要求（强制）
- 封面页：≤50字
- 每个内容页：≤200字
- 总页数：至少5-7页
- 每个要点必须充分展开：定义+用法+例子+注意事项

#### 合规要求
- **必须包含**：完整的风险提示（见SKILL.md第143-145行）
- **禁止出现**："保证收益""必定赚钱""零风险"等词汇

### 改写原则（重要）
1. **保持核心观点不变**
2. **转换为口语化表达**
3. **结构化为多页格式**
4. **添加合规风险提示**
5. **每个要点充分展开**

### 输出格式
```markdown
### 笔记标题（5个版本，每个≤20字）
版本1: [标题]
版本2: [标题]
...

#### 封面页
[内容，≤50字]

#### 笔记页一、[标题]
[内容，≤200字，充分展开]

[后续页面...]

#### 结尾推广
[固定推广内容]

### 正文
[独立文案]

**风险提示**: [完整风险提示]

> #标签1 #标签2
```

### 何时读取
Phase 2开始前读取，了解详细的小红书风格要求和输出格式

## 3. baoyu-xhs-images（小红书图文生成器）

### Skill名称
`baoyu-xhs-images`

### 路径配置
见 [sub-skills-config.md](sub-skills-config.md)

**集成位置**：`skills/baoyu-xhs-images/`

### 功能
生成小红书图文系列，支持10种视觉风格和8种布局方式。

### 核心能力

#### 10种视觉风格
- `cute`（默认）- 甜美可爱，经典小红书风格
- `fresh` - 清新自然
- `warm` - 温馨友好
- `bold` - 高冲击力
- `minimal` - 极简优雅
- `retro` - 复古怀旧
- `pop` - 活力醒目
- `notion` - 极简手绘线条，知性
- `chalkboard` - 彩色粉笔黑板风
- `study-notes` - 真实手写照片风

#### 8种布局
- `sparse`（默认）- 极简信息
- `balanced` - 标准内容布局
- `dense` - 高信息密度
- `list` - 枚举排名格式
- `comparison` - 左右对比布局
- `flow` - 流程时间线布局
- `mindmap` - 中心辐射思维导图
- `quadrant` - 四象限/圆形分区布局

#### 3种内容策略
- **Strategy A** - 故事驱动型（情感共鸣）
- **Strategy B** - 信息密集型（价值优先）
- **Strategy C** - 视觉优先型（氛围感）

### 执行流程
1. **Step 0**: 加载偏好设置（EXTEND.md）
2. **Step 1**: 分析内容 → analysis.md
3. **Step 2**: 确认 1 - 内容理解 ⚠️
4. **Step 3**: 生成 3 种策略方案
5. **Step 4**: 确认 2 - 方案/风格/元素选择 ⚠️
6. **Step 5**: 生成图像（逐张）
7. **Step 6**: 完成报告

### 输出格式
```
xhs-images/{topic-slug}/
├── source-{slug}.{ext}
├── analysis.md
├── outline-strategy-a.md
├── outline-strategy-b.md
├── outline-strategy-c.md
├── outline.md
├── prompts/
│   ├── 01-cover-[slug].md
│   └── ...
├── 01-cover-[slug].png
├── 02-content-[slug].png
└── NN-ending-[slug].png
```

### 支持修改
- 编辑：更新提示词 → 重新生成
- 添加：指定位置 → 创建提示词 → 生成
- 删除：删除文件 → 重新编号

### 何时读取
Phase 3 开始前读取，了解风格、布局、策略选项
详细风格定义：见 skill 中的 `references/presets/<style>.md`

## 4. xiaohongshu-viral-analyzer（小红书爆款分析器）

### Skill名称
`xiaohongshu-viral-analyzer`

### 路径配置
见 [sub-skills-config.md](sub-skills-config.md)

### 功能
基于流量黑客视角评估笔记爆款潜力，从四维结构分析内容质量，提供评分（0-100分）和优化建议（≤100字）。

### 核心分析框架

#### 高密四维结构
1. **极致对标**（25分）
   - 是否重组了经过市场验证的成功要素
   - 是否有清晰的赛道定位
   - 是否有跨界降维的创新

2. **情绪放大**（25分）
   - 是否直击人性（恐惧/贪婪/虚荣/色欲）
   - 是否制造反差、阶层焦虑或爽感
   - 是否每300字出可截图金句

3. **算法迎合**（25分）
   - 标题是否有视觉锤，0.5秒内拦截手指
   - 是否预埋槽点骗取互动和推荐
   - 是否SEO友好，关键词布局合理

4. **工业结构**（25分）
   - 是否符合标准模组（钩子→痛点→方案→召唤）
   - 黄金三秒：首句是否定生死
   - 正文是否短句分割，视觉降噪

### 输出格式
```markdown
## 小红书爆款分析报告

### 综合评分
<总分>/100分

### 四维得分
- 极致对标：<得分>/25分
- 情绪放大：<得分>/25分
- 算法迎合：<得分>/25分
- 工业结构：<得分>/25分

### 优化建议（≤100字）
<简洁的改进建议，指出1-2个关键提升点>
```

### 何时读取
Phase 2完成后，Phase 3开始前读取，评估小红书笔记的爆款潜力并提供优化建议。

## 调用顺序

```
用户输入
   ↓
Phase 0: 模式判断
   ↓
Phase 0.5: 账号风格选择
   ↓
Phase 1: content-creation-framework
   ├─ 改写模式：分析+优化
   └─ 创作模式：需求+选题+创作
   ↓
   ✋ 用户确认
   ↓
Phase 2: xiaohongshu-finance-writer
   └─ 改写为小红书风格（保持核心观点）
   ↓
   ✋ 用户确认
   ↓
Phase 2.5: xiaohongshu-viral-analyzer
   └─ 爆款潜力评分+优化建议
   ↓
Phase 3: baoyu-xhs-images
   └─ 生成小红书图文系列（每页一张）
   ↓
   ✋ 用户确认
   ↓
最终输出
```

## 关键要点

### Phase 1 → Phase 2 连接
- **传递内容**：Phase 1的改写/创作结果
- **核心要求**：Phase 2必须保持Phase 1的核心观点不变
- **风格转换**：专业表达 → 口语化小红书风格

### Phase 2 → Phase 2.5 连接
- **传递内容**：Phase 2的完整小红书笔记稿（标题+笔记图+正文）
- **评估维度**：极致对标、情绪放大、算法迎合、工业结构
- **输出结果**：综合评分（0-100）+ 优化建议（≤100字）

### Phase 2.5 → Phase 3 连接
- **传递内容**：Phase 2的小红书笔记稿 + Phase 2.5的评分和建议
- **提取关键**：从笔记稿中提取适合可视化的核心观点
- **风格统一**：信息图与笔记稿保持一致的专业度

### 用户确认机制
- **Phase 1确认**：展示改写/创作结果预览
- **Phase 2确认**：展示完整小红书笔记稿预览
- **Phase 3确认**：展示信息图生成结果预览

## 常见问题

### Q: Phase 2改写后观点变了怎么办？
**A**: 重新执行Phase 2，强调"保持核心观点不变"的要求

### Q: Phase 3生成的图像风格不统一？
**A**: 检查是否严格遵守了固定参数，重新生成

### Q: 用户在确认点要求大幅修改？
**A**: 重新执行当前Phase，融入修改意见，再次确认
