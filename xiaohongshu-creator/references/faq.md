## 模式判断相关

### Q1: 如何判断应该使用创作模式还是改写模式？

**判断依据**：输入内容的长度

- **创作模式**：输入 ≤ 200字
  - 适用场景：用户提供简短话题、关键词、需求描述
  - 任务特点：从零开始创作，需要选题、结构设计、内容创作
  - 示例："帮我写一篇关于ETF定投的小红书笔记"

- **改写模式**：输入 > 200字
  - 适用场景：用户提供已有文章、草稿、笔记
  - 任务特点：分析现有内容，提取核心观点，优化改写
  - 示例：用户粘贴了一篇300字的ETF介绍文章

### Q2: 改写模式后可以选择润色吗？

**可以**。在Phase 1确认点1，用户可以选择：

**选项A**：先润色内容，提升文字表达
- 优化句式表达
- 提升文字流畅度
- 增强可读性
- 保持核心观点不变

**选项B**：直接改为小红书风格
- 跳过润色
- 直接进入Phase 2

## 账号风格相关

### Q3: 如何选择合适的账号风格？

**选择决策树**：

```
官方账号/员工账号？
├─ 是 → 教育导向为主？→ 小红书主号-投教笔记
│        └─ 开户推广为主？→ 小红书主号-广告笔记
└─ 否 → KOS知识分享账号？
         ├─ 小白友好、口语化 → 叮叮猫的财经笔记
         ├─ 系统化、技术流 → 证券人排雷哥笔记
         ├─ 热点驱动、快节奏 → 申万宏源乐乐笔记
         └─ 理性分析、深度 → 仿中金公众号文风
```

**6个账号风格快速对比**：

| 账号名称 | 类型 | 风格特征 | 适用场景 | 语言风格 |
|---------|------|---------|---------|---------|
| **小红书主号-投教笔记** | 官方账号 | 理财教育导向，亲切口语化 | 知识科普、理财入门 | "攻略""秘籍""宝子们" |
| **小红书主号-广告笔记** | 官方账号 | 开户推广导向，利益吸引 | 新客权益、开户引导 | "薅羊毛""手慢无""限时" |
| **叮叮猫的财经笔记** | KOS账号 | 金融知识科普，小白友好 | 基础知识、概念讲解 | "宝子们""谁还在""速存" |
| **证券人排雷哥笔记** | KOS账号 | 系统化知识科普 | ETF、K线等技术知识 | 系统化、固定结尾 |
| **申万宏源乐乐笔记** | KOS账号 | 热点驱动型科普 | 市场热点、新规解读 | "新高""上车""新规" |
| **仿中金公众号文风** | 公众号风格 | 理性分析，长期主义 | 深度分析、投资理念 | 数据支撑、理性表达 |

**详细配置**：见 `references/account-styles-config.md`

### Q4: 用户想要切换账号风格怎么办？

**处理方式**：

1. 说明当前风格和目标风格的差异
2. 读取目标账号的风格配置文件
3. 重新执行Phase 2，应用新风格
4. 展示新风格的结果预览
5. 等待用户确认

**注意**：
- 不需要重新执行Phase 1
- Phase 1的核心观点保持不变
- 只改变表达方式、语言风格、结尾模板

### Q5: 如何添加新的账号风格？

**步骤**：

1. 在 `references/account-styles/` 目录创建新的配置文件
   - 文件命名：`[账号名].md`
   - 例如：`new-account.md`

2. 配置文件结构（参考现有账号风格文件）：
   ```markdown
   # 账号名称 - 风格配置

   **账号定位**：<账号定位描述>

   ## 风格定义
   <核心风格、思维模式、读者定位、互动模式>

   ## 核心特征
   <恒定特征、重要特征、可选特征>

   ## 语言风格
   <常用表达、句式特点>

   ## 内容结构
   <通用结构模式、进展节奏>

   ## 内容要求
   <篇幅标准、每页内容要求>

   ## 账号专属元素
   <标识信息、引流方式、结尾推广模板>
   ```

3. 更新 `references/account-styles-config.md`，添加新账号到列表

4. 更新主SKILL.md的Phase 0.5账号列表（如果需要）

## Phase执行相关

### Q6: Phase 1文本量不够800字怎么办？

**问题**：初稿文本量少于800字，不满足800-1500字要求

**解决方法**：

1. **检查要点展开是否充分**
   - 每个要点是否有：定义+用法+例子+注意事项
   - 补充更多细节说明

2. **补充案例和数据**
   - 添加具体案例
   - 增加数据支撑
   - 提供真实场景应用

3. **扩展投资转向部分**
   - 深化转向逻辑
   - 增加类比和说明
   - 提供更多实用建议

4. **增加实用建议**
   - 补充操作步骤
   - 添加注意事项
   - 提供进阶技巧

**检查标准**：
- 初稿必须 ≥ 800字
- 推荐范围：800-1500字
- 确保有足够的要点和金句供Phase 2提取

### Q7: Phase 2改写后核心观点变了怎么办？

**问题**：改写后的小红书笔记稿与Phase 1的核心观点不一致

**解决方法**：

1. **重新执行Phase 2**
   - 强调"保持核心观点不变"的要求
   - 对比Phase 1和Phase 2的输出
   - 找出变化的具体点

2. **修正改写稿**
   - 恢复被改变的核心观点
   - 保持原意的同时优化表达
   - 确保投资转向要素保留

3. **检查质量**
   - 核心观点是否完全一致
   - 表达是否符合小红书风格
   - 是否保持了知识价值

**预防措施**：
- Phase 2执行前明确强调保持核心观点
- 改写时定期对比原文
- 质量检查时重点检查核心观点

### Q8: Phase 2.5评分低于60分怎么办？

**问题**：爆款潜力评分低于60分，说明内容质量不高

**分析评分短板**：

**极致对标低**（<15分）：
- 缺乏对标爆款要素
- 创新不足
- 解决：增加爆款要素，参考成功案例

**情绪放大低**（<15分）：
- 缺乏情绪刺激
- 平铺直叙
- 解决：增加情绪钩子、制造反差、植入金句

**算法迎合低**（<15分）：
- 标题不够吸睛
- 缺乏互动槽点
- 解决：优化标题、预埋槽点、SEO优化

**工业结构低**（<15分）：
- 结构混乱
- 节奏不当
- 解决：调整结构、优化节奏、强化黄金三秒

**优化流程**：

1. 仔细阅读优化建议
2. 针对短板维度进行重点优化
3. 重新执行Phase 2，融入优化建议
4. 再次评分，直到达到合格标准

### Q9: Phase 3生成的图片风格不统一？

**问题**：多页信息图的风格不一致，影响整体效果

**检查清单**：

1. **背景色是否统一**
   - 所有图片必须使用：#FAF9F6（Warm Ivory）
   - 不允许使用渐变或多色背景

2. **主色调是否统一**
   - 只能使用3种莫兰迪色：#9FA8DA、#A5D6A7、#EF9A9A
   - 检查是否有其他颜色混入

3. **线条参数是否统一**
   - 宽度：2.5像素
   - 颜色：#757575
   - 透明度：85%
   - 虚线模式：8像素实线 + 6像素空白

4. **文字对齐是否统一**
   - 必须全部左对齐
   - 禁止居中对齐

5. **布局参数是否统一**
   - 模块宽度：70%
   - 间距：5%
   - 标题大小：画布高度的6%
   - 正文大小：画布高度的2.5%

**解决方法**：

1. 重新生成不统一的图片
2. 使用相同的风格提示词
3. 强调"风格一致性"要求
4. 生成前逐项检查风格参数

## 投资转向相关

### Q10: 投资转向如何做到自然不生硬？

**核心原则**：
- 软性引导，不是硬广
- 有知识价值，不是纯推广
- 从用户话题自然过渡

**5种转向策略**：

1. **理财观念转向**
   - 适用：购物、消费、生活方式类话题
   - 逻辑：从消费观念延伸到理财观念
   - 示例：美食话题 → "会吃也要会理财，餐费预算也是一种理财"

2. **财务安全转向**
   - 适用：职场、收入、生活规划类话题
   - 逻辑：从风险管理的角度切入投资
   - 示例：职场话题 → "收入上涨了，财务安全跟上"

3. **机会成本转向**
   - 适用：教育、投资、决策类话题
   - 逻辑：从成本收益角度分析
   - 示例：教育话题 → "教育也是投资，讲究投资回报率"

4. **资产配置转向**
   - 适用：宏观经济、市场趋势类话题
   - 逻辑：从资源配置角度展开
   - 示例：旅游话题 → "旅游也是一种体验型资产配置"

5. **财富增长转向**
   - 适用：创业、副业、收入增长类话题
   - 逻辑：从财富积累角度引导
   - 示例：副业话题 → "多渠道收入，财富加速增长"

**软性引导技巧**：

1. **从用户话题切入**
   - 先聊用户关心的话题
   - 建立共鸣和信任

2. **找到连接点**
   - 用类比或案例找到连接点
   - "就像...一样"的类比句式

3. **自然过渡**
   - 用过渡句连接
   - "说到这个，其实..." "这让我想到..."

4. **提供知识价值**
   - 给出实用的理财知识
   - 不是纯广告

5. **软性引导**
   - "如果你想深入学习..." "想要了解更多..."
   - 而非"快来开户"硬广

**详细指南**：见 `references/investment-pivot-guide.md`

### Q11: 什么话题都可以转向投资吗？

**可以**，但需要：

1. **找到合理的连接点**
   - 不是所有话题都适合所有转向策略
   - 需要分析话题特点，选择最合适的转向方式

2. **确保有知识价值**
   - 转向后必须提供有价值的理财知识
   - 不能为了转向而转向

3. **保持自然不生硬**
   - 转向逻辑要顺畅
   - 不能让读者觉得突兀

4. **避免强行转向**
   - 如果实在找不到合适的连接点
   - 可以选择不转向，或选择其他角度

## 输出质量相关

### Q12: 用户觉得不够专业（申万宏源官方风格）

**问题**：使用官方账号风格，但用户认为内容不够专业

**检查要点**：

1. **专业术语使用**
   - 是否使用了准确的专业术语
   - 术语解释是否清晰

2. **数据支撑**
   - 是否有具体数据支撑观点
   - 数据来源是否可靠

3. **市场分析**
   - 是否有对市场的分析
   - 分析是否深入

4. **风险评估**
   - 是否提及了投资风险
   - 风险提示是否完整

**优化方法**：

1. **增加数据和市场分析**
   - 补充最新市场数据
   - 添加行业分析
   - 引用权威报告

2. **使用更专业的表达**
   - 避免过于口语化
   - 使用行业标准表达
   - 减少网络流行语

3. **强化专业深度**
   - 增加技术分析
   - 提供专业见解
   - 展示专业知识

4. **明确引流开户**
   - 强化申万宏源的专业形象
   - 突出大券商的优势
   - 明确引导开户

### Q13: 笔记图页数太少或太多？

**问题**：笔记图页数不符合5-7页的标准

**页数太少**（<5页）：
- 问题：内容不够充分
- 解决：
  1. 补充更多要点
  2. 每个要点更充分展开
  3. 增加案例和注意事项

**页数太多**（>7页）：
- 问题：信息过载，用户难消化
- 解决：
  1. 合并相似要点
  2. 精简次要内容
  3. 保留核心干货

**标准页数结构**：
- 封面页：1张
- 内容页：3-5张（至少）
- 尾页：1张
- **总计：5-7页**

### Q14: 标题不够吸睛怎么办？

**问题**：生成的标题版本不够吸引点击

**优化技巧**：

1. **数字型标题**
   - "3个技巧..."
   - "5步学会..."
   - "90%的人不知道..."

2. **疑问型标题**
   - "为什么...?"
   - "谁还在...?"
   - "你真的懂...吗?"

3. **对比型标题**
   - "月薪3000 vs 月薪30000的理财差异"
   - "这样投资 vs 那样投资"

4. **利益型标题**
   - "解锁...新技能"
   - "薅羊毛！..."
   - "限时！..."

5. **恐吓型标题**
   - "再不...就晚了"
   - "...的致命错误"

**账号风格差异**：
- **官方账号**：专业+吸睛，平衡可信度和吸引力
- **KOS账号**：可以更大胆、更吸睛
- **叮叮猫**：疑问式、吸睛式为主

## 技术问题相关

### Q15: 子skill调用失败怎么办？

**问题**：Phase执行时，子skill无法正常调用

**检查清单**：

1. **skill名称是否正确**
   - content-creation-framework
   - xiaohongshu-finance-writer
   - baoyu-xhs-images
   - xiaohongshu-viral-analyzer

2. **skill是否已安装**
   - 检查skills目录
   - Windows: `C:\Users\<用户名>\.claude\skills\`
   - macOS/Linux: `~/.claude/skills/`

3. **路径配置是否正确**
   - 检查 `references/sub-skills-config.md`
   - 路径是否指向正确的SKILL.md文件

4. **skill文件是否完整**
   - 检查SKILL.md文件是否存在
   - 检查YAML frontmatter是否完整

**解决方法**：

1. **使用自动查找**
   - 确保skill安装在标准位置
   - 配置文件中使用 `{auto-detect}`

2. **使用自定义路径**
   - 如果skill不在标准位置
   - 在sub-skills-config.md中配置绝对路径

3. **重新安装skill**
   - 如果skill文件损坏
   - 重新复制skill文件到正确位置

**详细配置**：见 `references/sub-skills-config.md`

### Q16: 账号风格配置文件读取失败？

**问题**：无法读取账号风格配置文件

**检查清单**：

1. **文件是否存在**
   - 路径：`references/account-styles/<账号名>.md`
   - 检查文件名是否正确

2. **文件名是否正确**
   - zhukan-toujiao.md（主号投教）
   - zhukan-guanggao.md（主号广告）
   - dingdingmao.md（叮叮猫）
   - peileige.md（排雷哥）
   - lele.md（乐乐）
   - zhongjin-fengge.md（中金风）

3. **文件格式是否正确**
   - 必须是.md格式
   - 必须包含完整的配置结构

**解决方法**：

1. 检查文件路径和文件名
2. 如果文件不存在，从备份恢复
3. 确保文件编码为UTF-8

### Q17: 图像生成提示词无效？

**问题**：使用Phase 3生成的提示词无法生成预期效果

**检查清单**：

1. **提示词是否完整**
   - 包含所有风格参数
   - 背景色、主色调、线条参数等

2. **参数是否正确**
   - 背景色：#FAF9F6
   - 主色调：#9FA8DA #A5D6A7 #EF9A9A
   - 线条：2.5px, #757575, 85% opacity

3. **内容描述是否清晰**
   - 是否包含具体的页面内容
   - 是否指定了信息图类型

**优化方法**：

1. 增加具体的视觉描述
2. 明确信息图类型
3. 强调风格一致性

**详细指南**：见 `references/phase3-multi-image-guide.md`

## 其他问题

### Q18: 用户在某个确认点要求大幅修改怎么办？

**处理方式**：

1. **理解用户需求**
   - 明确用户不满意的具体点
   - 了解用户的期望

2. **重新执行当前Phase**
   - 融入用户的修改意见
   - 调整执行策略

3. **再次展示结果**
   - 展示修改后的结果
   - 说明做了哪些调整

4. **再次等待确认**
   - 确保用户满意后再继续

**注意**：
- 不得跳过用户确认
- 必须充分理解用户需求
- 必要时可以回溯到之前的Phase

### Q19: 如何确保金融合规？

**检查清单**：

1. **风险提示必须完整**
   ```
   投资有风险，入市需谨慎。以上内容仅供参考，不构成投资建议或收益承诺。关于相关权益，请以购买页面所展示的信息为准。如需了解更多详细规则，欢迎添加企业微信进行咨询。
   ```

2. **禁止出现的词汇**
   - ❌ "保证收益"
   - ❌ "必定赚钱"
   - ❌ "零风险"
   - ❌ "稳赚不赔"

3. **收益表述**
   - 可以说："历史年化收益率xx%"
   - 不可以说："未来收益保证xx%"

4. **产品介绍**
   - 客观描述产品特点
   - 不夸大收益
   - 明提示风险

### Q20: 如何跟踪和优化后续内容？

**发布前**：
1. 确认所有信息准确
2. 检查所有链接有效
3. 准备好回复话术

**发布时**：
1. 根据爆款评分选择最佳标题
2. 选择用户活跃时段发布
3. 准备好首条评论引导互动

**发布后**：
1. 关注前24小时数据（点赞、收藏、评论、转发）
2. 收集用户问题和反馈
3. 根据数据优化后续内容
4. 用用户问题作为后续选题

**长期优化**：
1. 建立内容数据表格
2. 分析爆款内容的共同特征
3. 不断迭代优化创作策略
4. 形成自己的爆款方法论
